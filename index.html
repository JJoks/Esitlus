Things to be known about Android Wear before diving in

Android Wear is in a nutshell a stripped down version of android with many key differences.

  • Wear devices are usually meant to be used as a companion device to another android device.
  • The design guidelines are completely different for Wear than they are on a typical android device.
  • Do not expect Android Wear to behave same way regular Android does, it is a completely different user experience and apps must be developed to accommodate for that.

What kind of applications are best suited for Wear

  • When building an application for wear it is good to consider whether it is more convenient to use such an application on a watch or on a phone/tablet.
  • Applications that simply need to notify someone should be developed for a phone as all notifications are sent to paired Android Wear devices.

    • Things you should not do

    • Do not port applications that are inconvienent or hard to use on a wearable device.
    • When porting layouts, do not use android elements like actionbars and tabs as they are not intented for use on Wear.

    Things you should do

    • If using a custom layout, use the currently unoffical but recommended Wearable UI library by Google.
    • When porting layouts, do not use android elements like actionbars and tabs as they are not intented for use on Wear.

    Using Wear layouts

    For the alternative Wear layouts to be used by wearables, you must create a watch stub.

    
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    final WatchViewStub stub = (WatchViewStub) findViewById(R.id.watch_view_stub);

    stub.setOnLayoutInflatedListener(new WatchViewStub.OnLayoutInflatedListener() {
        @Override
        public void onLayoutInflated(WatchViewStub stub) {
            mTextView = (TextView) stub.findViewById(R.id.text);
        }
    });
}
