SVG and animtion

1. To use an SVG as an <img>

Advantages: simplicity.

Disadvantages: the method doesn't give us advantages of SVG-files.

2. To use SVG as a CSS background-image


 .circule {
     background-image: url(./examples/assets/circule.svg);
 }

Advantages: simplicity.

Disadvantages: the method doesn't give us advantages of SVG-files.

3. To use inline SVG images.

width="130" height="130" viewBox="0 0 130 130"

width="130" height="130" viewBox="0 0 260 260"

width="130" height="130" viewBox="20 20 130 130"

Advantages: full interaction with styles of image.

Disadvantages: to loooooooooong code.

4. To use an SVG inline by means of teg <use>.

Advantages: full interaction with styles of image.

Disadvantages: additional teg.

5. To use an SVG as an <object>.

Advantages: possibility to interact in JS-code via Id.

Disadvantages: no way to apply style properties via CSS or HTML

6. To use SVG as an <iframe>

Advantages:
Supported in all major browsers.
Possibility to interact in JS-code via Id.

Disadvantages:
Bad for site's Search Engine Optimization (SEO).
SVG images added with this format are not scalable.
Have no advantages of SVG-file.

7. to use SVG as an <embed>

Advantages: just one more way.

Disadvantages: properties of svg-image can not be changed even by means of js and most modern browsers have deprecated and removed support for browser plug-ins.
