What is accessibility?

W3.org

The Web is fundamentally designed to work for all people, whatever their hardware, software, language, culture, location, or physical or mental ability.

Remember the old days?

  • Flash and screen readers
  • Flash and keyboard navigation
  • Flash and SEO

Now we write JavaScript. Hooray!
How do we not repeat the same mistakes?

  • Flash accessibility required more effort. Sometimes due to the nature of the "experience" (time-based and dynamically changing, thus too fast for a screen reader to read aloud in time), accessibility wasn't an option.
  • Browser inconsistencies with keyboard navigation were a bitch
  • Pages with no downlevel content were problematic for SEO. But even Google's crawling ability was hindered if there was no live text in your Flash movie (embedded in images or vectors)

Content is more than what‘s visible

  • Semantic markup
  • Alt text
  • Form labels
  • Offscreen text

What else?

  • Headings
  • Empty anchors bad!
  • Icon fonts in CSS pseudo-elements are the way to go for display-only things

Skip Links

Skip Links

They‘re useful for everyone.

 
$('.skip-links').find('a').on 'focus blur click', ($event) ->
  
  $thisLink = $(this)
  
  $thisLink.css({'left':'0'}) if $event.type == 'focus'
  $thisLink.css({'left':'-999999px'}) if $event.type == 'blur'
  
  if $event.type == 'click'
    $event.preventDefault()
    $thisLink.attr('href').focus()

$('a.nav-item-primary').on 'click', ($event) ->
  $event.preventDefault()
  
  $navSecondary = $(this).next('.nav-secondary')
  $accessLabel = $(this).children('.offscreen')
  
  if $navSecondary.hasClass('open')
    $navSecondary.removeClass('open')
    $accessLabel.text(' expand')
  
  else
    $navSecondary.addClass('open')
    $accessLabel.text(' collapse')
